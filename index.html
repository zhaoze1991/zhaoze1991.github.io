Binary Tree Maximum Path Sum

发表于 2015-09-01

Binary Tree Maximum Path Sum
我们来看一个完整点儿的例子
    1
   / \
  2  3
 / \  |
4  5 1
在这样一棵二叉树中，path-sum 等于 5 + 2 + 1 + 3 + 1 路径每次从底向上走，然后再向下走，向下了就不能再往上走了.
helper(TreeNode root)表示以root为根结点，能得到的最大pathsum
因为这个max可能再遍历过程中产生的，所以我们把max设置为全局变量。
当遍历到4时，pathsum = 4, 到2时，pathsum ＝ 2＋4＋5 ＝11
因此我们在更新max的时候，注意max ＝ {left + right + root.val, left + root.val, right + val}
大牛们给了条活路， 在计算left/right 的时候和0最比较，这样计算max比较方便。
因为我们计算的pathsum是以root为根的，所以左和右必须穿过根。在计算max的时候必须得加上根的值。

Best Time to Buy and Sell Stock

Best Time to Buy and Sell Stock
画画图。股票价格可能呈现一个没啥规律的折线。
用一遍变量min来记录0～i之间的最小值，那么 profit ＝ prices[i] - min就是在i点能获得的最大利润，对profit做取最大值操作。

Best Time to Buy and Sell Stock II
举个栗子，对于序列[5,1,2,3,4,1]
对于上升序列，[1,2,3,4] 我们可以这么买：

  1. 1买入，4卖出， 收益是3
  2. 1买入，2卖出，2买入，3卖出，3买入，4卖出，收益也是3

所以我们可以扫描一边数组。然后累加profit即可。有点像贪心法。

Best Time to Buy and Sell Stock III
这题比较好。
设left[i] 为0～i之间，交易1次能得到的最大利润，设right[i]为i～n之间交易能得到的最大利润。
那么最大profit = max{profit, left[i] + right[i]}
如何求left[i] 和right[i]?
对于left[i],我们定义一个左边的最小值，

left[i] = max(left[i - 1], prices[i] - min)

对于right[i],我们定义一个右边的最大值，

right[i] = max(right[i + 1], max - prices[i]);
