Coins in a Line
一排硬币，按次序拿，每次拿一枚或者两枚，谁拿到最后的一枚硬币谁就赢。注意，如果n＝0，先拿的的人也没拿到最后一枚，所以应该返回false。
我们来定义子问题dp[i]，表示有i枚硬币的时候，先拿的人是否赢。
我们先来看初始化条件，如果只有一枚或者两枚硬币，那么第一个拿的人肯定赢，如果有三枚硬币，第一个人稳输，如果没有硬币，算第一个人输。
我们可以得到这样的关系式
dp[i] = (dp[i-2] && dp[i-3]) || (dp[i-3] && dp[i-4])
啥意思呢，如果A拿了第i枚硬币，那么对手B可能拿i-1，或者拿了i-1和i-2枚，那么下一枚A能拿到的就是i-2或者i-3,那么我的策略就要求A不管下一枚拿到i-2或者i-3，都得能赢，否则，对手肯定会拿A不能赢的策略
如果A那了第i枚和i-1枚，那么对手B可能拿了i-2, 或者拿了i-2和i-3枚。那么下一枚A能拿到的就是i-3或者i-4，我们得保证i-3或者i-4都能赢，否则对手B肯定会做出不让A赢的拿法。
这里还用了一个哈希来记录已经计算过的拿法，否则这种指数级的计算肯定超时。

Max Tree

发表于 2015-09-08

Max Tree
维护一个递减的栈。

  1. 如果当前的值小于栈顶元素，则入栈。
  2. 如果当前的值大于栈顶的元素，开始做出栈操作，直到栈顶元素大于当前的值。新出栈的结点以旧出栈的结点为右子树。因为这是一个递减的栈，所以旧弹出的结点值比新弹出的要小，并且index比新弹出的要大。当前值的左子树，就是刚刚弹出的最后一个结点，因为它最大，然后再把当前值构成的树入栈。
  3. 最后，对栈做判断空操作，如果非空，则一直做2中的操作，新弹出的结点的右子树是旧弹出的结点。

Partition Array
双指针，begin和end。
在遍历数组的时候，分情况讨论:
1. 如果nums[begin] < k, 不用管，直接begin++
2. 如果nums[begin] >= k, 开始交换，但是交换的时候， 右边的end指针先移动，移动到第一个小于k的数字上
看了别人写的答案，觉得好多了。有点quick sort的思想。begin为第一个nums[begin] >= k的index， end 为第一个nums[end] < k的index，然后swap

Maximum Subarray II
left和right在做dp的时候，策略不同.
left[i] 表示，在[0~i]区间上，最大的sub-array, ending with i
right[i],表示，在[i~n-1]区间上，最大的sub-array

Minimum Subarray
没啥好说了，跟max差不多。

Maximum Subarray Difference
可能的解为:左边最小，减去右边最大，或者左边最大，减去右边最小
所以，思路跟Maximum Subarray II一样，区别在于多了两组变量

Subarray Sum
sum[index]表示[0~i]之间的和。
sum[index1] == sum[index2]， 则，在index1+1~index2之间的和为0.
用一个Map来记录index和sum，如果遇到两个index相同的sum时，表示index1 + 1 到index2的sum是0.
注意，需要添加一个虚拟index＝-1，这样我们才能记录index1=0的情况
举个栗子，
nums:   -3, 1, 2, -3
sum: 0, -3, -2, 0, -3

Subarray Sum Closest
题目给了O(nlogn)的时间复杂度要求，那么应该就是要排序了。
sum[i]表示在[0~i]上的和。如果i<j 那么sum[j]-sum[i] 表示nums[i+1]到nums[j]的和，如果sum[j]-sum[i]的值为最小，则就是解。
栗子
nums: -3, 1, 0, 1, -3, 5
sums: -3, -2, -2, -1, -4, 1
排序之后，－2和－2之间的diff为0， 表示nums[1+1] 到nums[2]之间的和为0
注意要加入一个dummy pair， 否则0那个位置就取不到了。
