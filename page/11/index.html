用heap来做的思路比较有意思，将所有list的head加入到最小堆中，然后每次从最小堆中取出最小的head，然后将head的next结点加入到最小堆中。设结点个数为n，时间复杂度O(nlog(k))

Generate Parentheses

发表于 2015-08-24

https://leetcode.com/problems/generate-parentheses/
经典递归，思路如下：
用left 和 right 表示左边和右边括号的数目，有如下几种边界条件：

  1. left < right, 表示产生的括号中，右边的括号多于左边的，e.g. “)” 这种情况not valid，直接返回
  2. left 和 right 均等于n，则表示这是一个符合条件的组合，输出。
  3. left < n， 则表示左边的括号还可以继续增加

e.g.
generateParenthesis(3);
((()))
(()())
(())()
()(())
()()()

ZigZag Conversion

发表于 2015-08-24

https://leetcode.com/problems/zigzag-conversion/
题目本身可能有啥数学解法，但是，我不喜欢数学解法。
做这题的时候挂在一个corner case上，如果numRows = 1, 则立即返回原来的字符串。
基本思路如下：
定义一个string类型的数组，数组长度为numRows，可以把这个数组看成是从上到下排列的。
然后开始遍历整个字符串，同时将字符填到数组中。数组的index在［0 ～ numRows］之间，根据向上移动或者向下移动来改变index的值。

发表于 2015-08-23

https://leetcode.com/problems/longest-substring-without-repeating-characters/
双指针题。维护一个window，设window左边为left, 右边为i， 不断向右移动，处理以下可能情况：

  1. 最简单的情况，如果字符没出现过，直接加入到哈希表中
  2. 如果该字符出现过，分两种情况处理:
    1. 该字符在当前维护的window左侧，更新哈希表即可
    2. 如果该字符出现在当前维护的window中，即 left <= map[character] < i, 则需要重新计算max 需要更新left的值

最后，得处理corner case, 就是如果在window中没有出现重复的字符，第十三行代码则不会被执行，所以在结尾处，得再计算一次max.
