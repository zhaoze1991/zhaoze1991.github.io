Maximum Product Subarray

有些蛋疼的面试官非得让你把空间减少到O(1), 先来看看空间O(1)的Maximum Subarray

在上面的代码中，我们用一个localmax的变量，记录了遍历过程中产生的max，然后每次把globalmax和这个localmax做对比即可。
同样，我们在求max product subarray的问题中，也可以用这样的思想。但是，乘法的max可能有两种，一种是 最大正数＊正数，另一种是 最小负数＊负数。于是。我们就用两个变量分别来记录。用localmax和localmin分别来记录遍历过程中产生的max和min。我们可以得到如下的动态规划方程：
localmax = max(nums[i], max(localmax*nums[i], localmin*nums[i]));
localmin = min(nums[i], min(localmin*nums[i], localmin*nums[i]));
最后再用一个globalmax 的变量来记录整个过程中产生的最大值。

Insertion Sort List

发表于 2015-09-04

Insertion Sort List
再次强调 这种可能变动head的题，要用dummy node，省时省力多了。
思路如下，用两根指针，一个head，一个prev，表示head之前的元素，分情况讨论：

  1. 如果head的值大于等于prev的值，就继续向后移动，prev指向head，head指向head.next
  2. 如果head的值小于prev的值，那我们就要把head插入到它应该在的地方。首先，得把head从当前位置给删掉，然后从头开始扫描，把它放在正确的位置。

Sort List
O(nlogn)的排序算法也就那么几个，对于这题，还是感觉merge sort比较趁手。
问题在于如何用O(1)的时间复杂度，去找到对应的元素？如果是数组，很好办，直接用index就能访问。但是对于链表，我们能想到的naive的方法就是每次去遍历，然后找到第k个元素。这样显然不符合题意。咋办？我们只能在递归过程中，对head做些手脚。在split过程中，如果当前递归区间的长度是1，即begin == end， 那么我们就把head断开，然后head向后移动一位，再返回断开之前的那个点。
这题没有一次AC，主要原因在于split的时候，左边的区间取的有问题，左边的end ＝ mid， 而不是mid－1， 查了下mergesort的算法，好像也是左边的end ＝ mid。

LRU cache

LRU cache
这题用一个双向的链表来做。如果一个元素被hit到，就把这个元素放到链表的头部。如果元素不存在，就添加。添加之前先检查cache的大小，如果cache满了，那么就删掉双向链表中尾部的元素，然后同时删掉哈希表中对应的元素。
做的时候又没有一次AC，原因在于开始写Unit的时候，没把key考虑进去，导致后来删除哈希表里的数据时，找不到key了。

Gas Station

发表于 2015-09-03

Gas Station
题目告诉我们解是唯一的。那么我们就从0开始走好了。走的过程中，更新tank的容量，如果tank的容量小于0，则表示当前的start不靠谱，我们得重新找start。如何更新start呢？我们可以得出start～i 之间的点都舍掉，start ＝ i ＋ 1
反正一下
如果gas[start] + … + gas[i] < cost[start] +…+ cost[i]
我们假设存在某一点x为可行的新的start，那么我们会得到：
gas[x] +…+gas[i] > cost[x] +…+cost[i]
因此我们可以得到gas[start] + … + gas[x-1] < cost[start]+…+cost[x-1], 根据这一不等式，我们无法到达x点。所以x点不存在。
注意 在走的过程中， 记录总的油量和耗油量。如果总油量小于总耗油量，则返回－1。
