Trapping Rain Water

发表于 2015-08-26

Problem Description




这种题就是属于不看答案就呵呵的。
如图所示，每个bar能顶住水，取决于这块bar两侧的bar是否都高于它，如果这块bar能够顶住水，这块bar能顶住水的容积为，两侧最小的高度－自己的高度，遍历累加每块bar能装的容积，就能得到答案。
如何求每块bar两侧最大的bar呢？
设两个数组，left和right，left[i]表示bar i左边最大的bar，right[i]表示bar i 右边最大的bar, 我们可以得到以下两个关系式：


left[i] = max(left[i - 1], height[i]);
right[i] = max(right[i + 1], height[i]);


得到每个bar对应的两边最大的bar之后，我们再从头到尾扫描一次数组，求能够累计的水容积。

First Missing Positive

发表于 2015-08-26

Problem Description
思路：
如果nums[i] 不在它应该在的位置上，那就把num[i] 替换到它应该在的位置上。在替换过程中，得考虑几个条件。

  1. 如果nums[i] <= 0, 或者nums[i] > 数组长度，那么跳过，因为数组中没有可以放置它的地方
  2. 如果nums[i] 和它 交换的位置的数相同，skip。否则就死循环了。e.g.［1， 1］

在交换完之后，i需要停留在原来的位置，所以i -= 1;

整个数组交换完之后，从头到尾扫描一次数组，第一个不在其位的数字就是题目想要的答案。

Problem Description
这题要求时间复杂度是O(logn), 能想到最简单的方法就是做一次二分查找， 找到一个target，然后向两边扩展，找到两端的分界。但是，如果数组中所有的数字都是相同的，那么这种方法的时间复杂度就是O(n)。
所以，这题可以做两次二分查找。

  1. 找到最左边的等于target的数字
  2. 找到最右边等于target的数字

注意代码中，当nums[mid] == target 时候的处理，如果找左边，则将mid赋值给end，右边反之。

Problem Description
用一个stack来记录“未配对”的括号的index， 分如下几种情况进行讨论：

  1. 如果当前字符是”(“, 直接入栈。
  2. 如果当前字符是”)”, 则对stack中顶部的元素进行分析，
    2.1 如果顶部元素是”)”, 则将当前元素的坐标入栈。

Substring with Concatenation of All Words

发表于 2015-08-25

solve this problem
虽然也是一个双指针题，但是我这种解法比较暴力，时间复杂度为O(n^2)
大致思路如下：
先将words遍历一次，记录所有string出现的次数。
在for循环中，遍历每一个可能的position，注意每次是i+1， 然后开一个新的哈希表，记为windows，意思是满足条件的一个windows。
在内层循环中，对每次得到的substring做处理，分如下几种条件：

  1. 该字符不在words中，则跳出循环，表示这样的windows不可能符合条件。
  2. 如果该字符在words中，并且在windows中未出现过，则加入windows即可。
  3. 如果该字符在windows中出现过，分两种情况讨论
    3.1 该字符出现的次数小于words中该字符的次数，则更新哈希表
    3.2 该字符出现的次数大于words中该字符的次数，跳出循环。

最后，将windows和words进行对比，如果windows里的内容与words一致，则更新结果，否则继续找。
