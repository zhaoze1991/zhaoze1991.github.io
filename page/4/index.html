Min Stack

发表于 2015-09-06

Min Stack
第一种方法，用一个stack，但是有一个min的指针，表示当前stack中最小的的数值。
在入stack的时候，先检测入栈的值是否小于或者等于min，如果是，则将min加入到栈中，更新min，然后再把值加入到栈中。
在出栈的时候，记录出栈的值为val，检测val是否等于min，如果是，那么把min更新为stack当前栈顶的值，再pop一次。

用两个栈，一个用来存储普通的入栈出栈操作，另一个min栈用来记录当前栈中最小的数字。
所以每次入栈时候，比较入栈的值和min栈顶的元素，将两者中较小的入栈
出栈时，两个栈同时出栈。

Text Justification
这题就是实现起来比较蛋疼。
我觉得这题有点维护一个window的意思，所以就用把变量的名字搞成left，表示当前window中最左边的一个单词的坐标。curLen表示当前window中所有单词的长度。注意，维护这个curLen的时候不要急于加入空格的长度，会搞的比较麻烦。所以我们在判断时候超过给定的最大长度时，有这样的关系式：curLen ＋ words[i] + (i - left) > maxWidth. 如果要加入第i个词，那么长度就变成了curLen+…如果这个长度大于maxWidth，那么我们就开始Build new String。

首先我们得知道一共剩下多少的空间拿来填充space，所以spaceCount ＝ maxWidth － curLen，然后我们得计算出一共有多少个slot可以查space。slot ＝ (i-1) - left.
计算出上述变量之后，我们还得对slot做判断。

  1. 如果一个slot都没发插，比如有一个超级长的词,那么我们直接在新的string中插入单词.
  2. 此时i==n（注意，最后一行可能有多个单词）我们就插入一个单词，然后跟一个空格，

对于1和2， 最后，我们再用空格把剩余空间补齐

  1. 如果我们有多个slot可以插空格，我们先来计算每个slot能插几个空格，each = spaceCount / spaceSlots， 但是这可能不能被整出，还剩下几个空格，咋办呢，再计算extra ＝ spaceCount ％ spaceSlots。我们在构建新的字符串的时候，先插入单词，然后添加空格，空格的长度为each ＋ 1 或者 ＋ 0， 取决于此时extra是否小于0，在循环中，每次对extra做减1的操作。

Maximum Product Subarray

有些蛋疼的面试官非得让你把空间减少到O(1), 先来看看空间O(1)的Maximum Subarray

在上面的代码中，我们用一个localmax的变量，记录了遍历过程中产生的max，然后每次把globalmax和这个localmax做对比即可。
同样，我们在求max product subarray的问题中，也可以用这样的思想。但是，乘法的max可能有两种，一种是 最大正数＊正数，另一种是 最小负数＊负数。于是。我们就用两个变量分别来记录。用localmax和localmin分别来记录遍历过程中产生的max和min。我们可以得到如下的动态规划方程：
localmax = max(nums[i], max(localmax*nums[i], localmin*nums[i]));
localmin = min(nums[i], min(localmin*nums[i], localmin*nums[i]));
最后再用一个globalmax 的变量来记录整个过程中产生的最大值。

Insertion Sort List

发表于 2015-09-04

Insertion Sort List
再次强调 这种可能变动head的题，要用dummy node，省时省力多了。
思路如下，用两根指针，一个head，一个prev，表示head之前的元素，分情况讨论：

  1. 如果head的值大于等于prev的值，就继续向后移动，prev指向head，head指向head.next
  2. 如果head的值小于prev的值，那我们就要把head插入到它应该在的地方。首先，得把head从当前位置给删掉，然后从头开始扫描，把它放在正确的位置。

Sort List

问题在于如何用O(1)的时间复杂度，去找到对应的元素？如果是数组，很好办，直接用index就能访问。但是对于链表，我们能想到的naive的方法就是每次去遍历，然后找到第k个元素。这样显然不符合题意。咋办？我们只能在递归过程中，对head做些手脚。在split过程中，如果当前递归区间的长度是1，即begin == end， 那么我们就把head断开，然后head向后移动一位，再返回断开之前的那个点。
