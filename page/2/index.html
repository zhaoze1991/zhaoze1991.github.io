Jump Game I, II

发表于 2015-08-27

Jump Game I
贪心法不好做啥总结，所以干脆就用dp来做，当然时间效率肯定不咋滴。
dp[i] 表示第i个位置是否可以到达。 我们可以有这样的关系是，
if dp[j] = true and nums[j] + j >= i, 则dp[i] = true;
首先初始化，dp[0] = true;
然后从i = 1开始遍历。注意，我们推断dp[i]时，从i - 1 开始推，恰巧没有超时。

Jump Game II
设dp[i]为跳到第i的位置需要的最小次数。我们可以得到这样的关系式：
dp[i] = min(dp[i], dp[j] + 1) j可以跳到i
跟jump game I 一样的做法，超时。

改一改。做法有点BFS的意思。
设dp[i]为跳到第i的位置需要的最小次数。
把从第i个点 能跳的范围看成一个window， 那么在这window中，每个位置需要的最小跳跃次数为dp[i] + 1
遍历这个window的过程中，找到下一次跳能到达的最远的点。这样时间复杂度就是O(n), 因为每个点被访问一次。

Maximum Subarray

发表于 2015-08-27

Problem Description
这题有好多种AC的方法。这里我写了两种。
dp[i]表示从0~i之间最大的subarray, 用一个local的数组，来记录局部最优。
那么dp[i] = max{dp[i-1], local[i]}
其中local[i] = max{local[i - 1] + nums[i], nums[i]}

感觉这种方法不能算DP，算贪心吧。用一个local数组，来记录，包含i的区间中最优的解。然后再用一个变量来记录最大的局部最优解，最后返回这个变量。

N-Queens I
1和2的代码基本相同。这种题，就属于是个人都会写，但是bug-free就不是那么简单了。
这次做我还是没有一次过。 蛋疼。
思路如下：
按列来放置皇后，现在第一列放置一个皇后，然后dfs到下一列去放置皇后，放置之前check当前位置能否放置。
在valid函数中，可以做一点优化，开一个数组occupy[n]，occupy[i]表示第i列，放置皇后的位置为occupy[i]，这样O(n)的时间复杂度就能把能放置的位置判断出来。

Pow(x, n)

发表于 2015-08-27

Problem Description
二分法。这篇blog加载了mathjax的javascript，果然反应就慢了很多。

  1. 首先判断x的值，如果 x ＝ 0 或者 x ＝ 1， 则直接返回。否则，如果x ＝ 0， n < 1， 1 / x 就挂了。
  2. 首先判断n的符号，如果n是负数，则 x = 1 / x， 然后再把n变成正数。
  3. 根据如下关系式进行二分:
    $$ x ^ 4 = (x ^ 2) * (x ^ 2), x ^ 8 = ((x ^ 2) ^ 2) ^ 2 $$

则，当 n 为偶数的时候， x ＝ x ^ 2, n /= 2; 当 n 为奇数的时候，n -= 1, ans *= x

Multiply Strings

发表于 2015-08-26

Problem Description
如果直接把字符串转成数字进行乘，肯定会溢出，所以只能一位一位来。这题就是传说中的“大数相乘”
自己写的每次都出bug，参考答案写的很巧妙，膜拜啊，背答案也要背这种简单好记的。
思路大致如下：

  1. 将字符串反转过来。因为我们构建的新的数组中，个位的index ＝ 0， 十位的index ＝ 1， 如果个位在后面，就不好做了。
  2. 用一个数组来存储每一位数的和。举个栗子，123 ＊ 123， 个位 ＝ 3 ＊ 3， 十位 ＝ 2 ＊ 3 ＋ 3 ＊ 2， 百位 ＝ 1 ＊ 3 ＋ 2 ＊ 2 ＋ 3 ＊ 1， 转换到数组的index上就是


    ans[1] = num1[0] * num2[1] + num1[1] * num2[0]

  3. 最后遍历一次数组，将结果输出。
