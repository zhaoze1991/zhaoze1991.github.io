Partition Array
双指针，begin和end。
在遍历数组的时候，分情况讨论:
1. 如果nums[begin] < k, 不用管，直接begin++
2. 如果nums[begin] >= k, 开始交换，但是交换的时候， 右边的end指针先移动，移动到第一个小于k的数字上
看了别人写的答案，觉得好多了。有点quick sort的思想。begin为第一个nums[begin] >= k的index， end 为第一个nums[end] < k的index，然后swap

Maximum Subarray II
left和right在做dp的时候，策略不同.
left[i] 表示，在[0~i]区间上，最大的sub-array, ending with i
right[i],表示，在[i~n-1]区间上，最大的sub-array

Minimum Subarray
没啥好说了，跟max差不多。

Maximum Subarray Difference
可能的解为:左边最小，减去右边最大，或者左边最大，减去右边最小
所以，思路跟Maximum Subarray II一样，区别在于多了两组变量

Subarray Sum
sum[index]表示[0~i]之间的和。
sum[index1] == sum[index2]， 则，在index1+1~index2之间的和为0.
用一个Map来记录index和sum，如果遇到两个index相同的sum时，表示index1 + 1 到index2的sum是0.
注意，需要添加一个虚拟index＝-1，这样我们才能记录index1=0的情况
举个栗子，
nums:   -3, 1, 2, -3
sum: 0, -3, -2, 0, -3

Subarray Sum Closest
题目给了O(nlogn)的时间复杂度要求，那么应该就是要排序了。
sum[i]表示在[0~i]上的和。如果i<j 那么sum[j]-sum[i] 表示nums[i+1]到nums[j]的和，如果sum[j]-sum[i]的值为最小，则就是解。
栗子
nums: -3, 1, 0, 1, -3, 5
sums: -3, -2, -2, -1, -4, 1
排序之后，－2和－2之间的diff为0， 表示nums[1+1] 到nums[2]之间的和为0
注意要加入一个dummy pair， 否则0那个位置就取不到了。

Largest Rectangle in Histogram
维护一个递增的栈，每次比较栈顶元素和当前元素，如果当前元素小于栈顶元素，则出栈。如果当前元素大于栈顶元素，则入栈。
我们来看个栗子:



2, 1, 5, 6, 2, 3

  1. push(2)
  2. 1 < 2, 表示index＝0的这个直方图的右边的边界到了，我们可以开始计算以它为高度的最大直方。那么这个宽度怎么算？i = 1, 2 出栈后，栈为空，宽度就是i到最左边（因为这是一个递增栈，如果栈为空，表示我们取出的当前的直方是最低的，它的宽度可以一直扩展到最左边。）假如栈不空，则宽度等于 i - s.peek() - 1(因为要减去s.peek() 这个直方，它比s.pop()要低，阻止了s.pop()这个直方向左扩展)
  3. 1, 5, 6 这是一个递增的序列，入栈
  4. 碰到2，2比6小，6出栈，width = 6的本身；5出栈，宽度是2; 2入栈
    5， 3入栈
  5. 这时候栈里面还剩1，2，3，这时候，我们的val ＝ －1(很巧妙的写法)，3出栈，宽度是1； 2出栈，宽度是4; 1出栈，宽度是6

Edit Distance

发表于 2015-09-06

Edit Distance
老头子上算法课反复讲了这题，我都能把代码背下来了。
dp[i][j] 表示A[0~i]转换成B[0~j]所需要的最少的次数。
初始化：

  1. dp[0][0] ＝ 0，2个字符串都为空，不需要转换
  2. dp[i][0] = i, 第二个字符串为空，转换需要i次
  3. dp[0][i] = i, 第一个字符串为空，转换需要i次

操作含义：

  1. dp[i-1][j-1]: A, B不算当前的字符，把A转成B需要最小次数
  2. dp[i-1][j]: A不算当前的字符，把A转成B需要的最小次数
  3. dp[i][j-1]: A转成B不算当前字符，所需要的最少次数
