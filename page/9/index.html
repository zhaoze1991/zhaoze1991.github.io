Minimum Window Substring

发表于 2015-08-28

早上看到这题就跳过了，觉得这么麻烦 考到概率低下。结果特么下午面试就来了这么道题。写了好多bug。WTF!!!
代码写的跟狗屎一样长，在于java的哈希表操作太麻烦，不能像c++那样直接加加减减。
思路如下：
首先，开一个哈希表，map 用来记录t中每个字符出现的次数。
再开一个哈希表，window，用来记录当前window中每个字符出现的次数。
还得有一个变量count来记录当前window中符合条件的字符的数量。
然后开始对字符串s做遍历。
如果字符c在哈希表map中，那么把这个字符c加入到当前的window中。注意，当window中c出现的次数小于map中c出现的次数，才去更新count。
如果count等于t的长度，意思是目前的window中，已经包含了所有t中的字符，那么 我们开始要做些事儿了去移动window的左侧了。
在第29行代码while循环中

  1. 得跳过不在map中的字符
  2. 如果字符在map中，而且字符出现的次数大于map中字符出现的次数，也得跳过。 e.g. s=aaaaabaa t=ab

这样，通过这个while循环，我们就得到了当前包含所有字符的最小的一个window，然后去更新min和结果。
完了之后，我们把left右移一位，使得window中不再包含所有的字符。

总之。太麻烦了。bug-free好难啊。这题的思想和Longest Substring Without Repeating Characters以及Substring with Concatenation of All Words类似。注意体会如何移动left。

Jump Game I, II

发表于 2015-08-27

Jump Game I
贪心法不好做啥总结，所以干脆就用dp来做，当然时间效率肯定不咋滴。
dp[i] 表示第i个位置是否可以到达。 我们可以有这样的关系是，
if dp[j] = true and nums[j] + j >= i, 则dp[i] = true;
首先初始化，dp[0] = true;
然后从i = 1开始遍历。注意，我们推断dp[i]时，从i - 1 开始推，恰巧没有超时。

Jump Game II
设dp[i]为跳到第i的位置需要的最小次数。我们可以得到这样的关系式：
dp[i] = min(dp[i], dp[j] + 1) j可以跳到i
跟jump game I 一样的做法，超时。

改一改。做法有点BFS的意思。
设dp[i]为跳到第i的位置需要的最小次数。
把从第i个点 能跳的范围看成一个window， 那么在这window中，每个位置需要的最小跳跃次数为dp[i] + 1
遍历这个window的过程中，找到下一次跳能到达的最远的点。这样时间复杂度就是O(n), 因为每个点被访问一次。

Maximum Subarray

发表于 2015-08-27

Problem Description
这题有好多种AC的方法。这里我写了两种。
dp[i]表示从0~i之间最大的subarray, 用一个local的数组，来记录局部最优。
那么dp[i] = max{dp[i-1], local[i]}
其中local[i] = max{local[i - 1] + nums[i], nums[i]}

感觉这种方法不能算DP，算贪心吧。用一个local数组，来记录，包含i的区间中最优的解。然后再用一个变量来记录最大的局部最优解，最后返回这个变量。
